# 🔥 CRUCIBLE

**Self-Improving RL Training Environment | OpenEnv Hackathon 2026**

> *"Fixed environments have ceilings. CRUCIBLE doesn't — because the hardest problem you face tomorrow is built from your failures today."*

---

## What It Is

CRUCIBLE is a three-agent RL training environment where the curriculum is generated by the agent's own failure history.

| Agent | Role | Objective |
|-------|------|-----------|
| **Executor** | Solves procurement compliance tasks | Score well |
| **Arbiter** | Scores every attempt, gives feedback | Score accurately (frozen) |
| **Architect** | Reads failures, generates next task | Keep Executor in 0.45–0.70 band |

The curriculum is not designed — it **emerges** from the Executor's failure history.

## The Three Phases

### Phase 1 — Baseline
```bash
python main.py baseline
```
Runs all static tasks with base model. Stores episode-zero scores. No training.

### Phase 2 — GRPO Training
```bash
python main.py train
```
50 episodes on static tasks. Arbiter reward → GRPO loop. Architect not active. Establishes failure profile.

### Phase 3 — Architect Activation
```bash
python main.py architect
```
Architect reads Phase 2 failure log. Generates dynamic tasks targeting weakest axis. Reward curve steepens.

---

## The Key Insight: The Learning Band

```
Score < 0.20  →  Too hard. AI produces noise. No learning.
Score 0.45–0.70  →  PRODUCTIVE LEARNING ZONE
Score > 0.90  →  Too easy. AI learns nothing new.
```

The Architect's only job is to keep the Executor inside the band **forever**.

## Reward Formula

```
final_reward = base_score
             + delta_bonus (learning from feedback, max +0.15)
             + coherence_bonus (±0.10 world state consistency)
             + malformed_penalty (-0.15 if empty/broken response)

[clamped to 0.0–1.0]
```
