## The Problem

Every existing RL training environment has a fixed curriculum. Once the agent masters the available tasks, learning stops — not because the agent hit its ceiling, but because the environment did.

## The Solution

CRUCIBLE is a closed three-agent loop where the training curriculum is generated by the agent's own failure history.

| Agent | Role | Objective |
|-------|------|-----------|
| **Executor** | Solves procurement compliance tasks | Score well |
| **Arbiter** | Scores every attempt across 5 axes | Score accurately (frozen) |
| **Architect** | Reads failures, generates next task | Keep Executor in band |

**The learning band:** Executor must score 0.45-0.70 for learning to happen.
- Below 0.20 -> too hard, no signal
- 0.45-0.70 -> productive learning zone ✓
- Above 0.90 -> too easy, no signal

The Architect's only job: keep the Executor in this band forever.

## Quick Start

```python
pip install git+https://huggingface.co/spaces/YOUR_USERNAME/crucible-env

from crucible_env.client import CrucibleEnv
from crucible_env.models import CrucibleAction

with CrucibleEnv(base_url="https://YOUR_USERNAME-crucible-env.hf.space").sync() as env:
    obs = env.reset()
    action = CrucibleAction(
        decision="NON-COMPLIANT",
        reasoning="SAM.gov expired, vendor ineligible per FAR 4.1102",
        violations_found=["FAR 4.1102 — expired SAM.gov registration"],
        confidence=0.9
    )
    result = env.step(action)
    print(f"Reward: {result.reward:.3f}")
```

## Reward Formula

```text
final_reward = base_score (weighted across 5 axes)
             + delta_bonus (learning from feedback, max +0.15)
             + coherence (±0.10 world state consistency)
             + malformed_penalty (-0.15 if broken response)
             [clamped 0.0–1.0]
```

Scoring weights: correctness 35%, completeness 25%, reasoning_transparency 20%, efficiency 10%, generalization 10%
